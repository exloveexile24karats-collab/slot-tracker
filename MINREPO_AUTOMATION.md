# min-repo 自動取得のセットアップ

これを設定すると、毎晩3時（日本時間）に自動で min-repo.com からプラザ本店IIのデータを取得し、
まだ登録されていない日付だけをFirestoreに追加します（既存のデータは一切上書きしません）。

## やること（1回だけ）

### ステップ1：各機種ページに「min-repo正式名称」を登録する

1. アプリを開き、自動取得したい機種のページ（タブ）を開く
2. 暗証番号でロックを解除する
3. ページ上部、機種名の下にある「min-repo正式名称（自動取得用・任意）」の欄に、
   min-repoでの正式な機種名を入力する
   - 一度でも「全体データ」タブでその日の機種別サマリーを保存していれば、
     プルダウン候補に正式名称が出てくるので、そこから選ぶと確実です
   - 例：「スマスロ 甲鉄城のカバネリ 海門決戦」

この欄を設定した機種だけが自動取得の対象になります（設定していないページには何もしません）。

### ステップ2：Firebaseのサービスアカウントキーを発行する

1. [Firebaseコンソール](https://console.firebase.google.com/) を開き、このプロジェクトを選択
2. 左上の歯車アイコン →「プロジェクトの設定」
3. 上部タブから「サービス アカウント」を選択
4. 「新しい秘密鍵の生成」ボタンをクリック
5. 確認画面で「キーを生成」をクリックすると、JSON形式のファイルがダウンロードされます
   （このファイルは他人に絶対に見せないでください。Firestoreへの書き込み権限を持つ鍵です）

### ステップ3：GitHubにそのキーをSecretとして登録する

1. GitHubのリポジトリページを開く
2. 「Settings」タブ →左メニュー「Secrets and variables」→「Actions」
3. 「New repository secret」をクリック
4. 以下のように入力：
   - Name: `FIREBASE_SERVICE_ACCOUNT_JSON`
   - Secret: ステップ2でダウンロードしたJSONファイルの中身を**そのまま全部**貼り付け
5. 「Add secret」をクリック

## 動作確認

1. GitHubリポジトリの「Actions」タブを開く
2. 左メニューから「Fetch min-repo data」を選ぶ
3. 右側の「Run workflow」ボタンから、今すぐ手動で1回実行できます
4. 実行結果（ログ）で、どの日付・どの機種のデータを追加したか確認できます

普段は何もしなくても、毎晩3時（日本時間）に自動で実行されます。

## 仕組みのポイント

- 取得対象は「プラザ本店II」のみです（別のホールを追加したい場合はスクリプトの
  `TAG_LIST_URL` を書き換える必要があります）
- 2026年5月15日より前の日付は取得しません（`fetch-minrepo.js` の `STOP_DATE` で変更可能）
- 既にデータがある日付（手入力・自動取得どちらも）には絶対に触れません
- 「機種別サマリー」「末尾別データ」は「全体データ」タブに自動保存されます
- 各機種の台番号ごとのデータは、min-repo正式名称を登録したページに自動保存されます
- イベント名（「旧イベント日」など）は自動反映されません。今まで通り「イベント登録」パネルで手動管理してください
